# Design Polish — 디자인 고도화 규칙의 단일 진실

> 원리: "더 창의적으로"는 평균으로 회귀한다. 작동하는 것은
> ① 생성 전에 내려진 구체적 결정(방향·시그니처) ② 금지 목록(평균 밖으로 미는 힘)
> ③ 레퍼런스×차용점 짝(무엇을 가져올지까지 지정).
> 검증: 2026-06-11 Travelnote 데모 2종(메인·예약)으로 실증 — 색 페어 전수 AA PASS, 토큰 규율 0건.

## 0. 입력 우선순위 (충돌 시 위가 이김)

1. PROJECT.md `## 금지 규칙` (기존 섹션)
2. PROJECT.md `## 디자인 방향` (v0.8 신규 섹션 — 없으면 이 스킬의 기본값만)
3. 이 스킬의 공통 금지 목록 (§2)
4. 설치된 취향 스킬 (high-end-visual-design, gpt-taste, design-taste-frontend 등)

**취향 스킬 차용 규칙 (고정)**: 취향 스킬의 구체 수치(px·hex·특정 폰트명)는 값 그대로 쓰지 않는다.
**조합 판단(구도·위계·여백 비율·모션 곡선)만 차용**하고 값은 반드시 토큰을 통과한다.
detect-hardcoded 훅이 src/screens/도 차단하므로(v0.8) 우회 시도 자체가 막힌다.
- 폰트: 취향 스킬들의 Inter 금지는 영문 기준. 한글 UI의 Pretendard는 토큰 강제 — slop 회피는
  폰트 교체가 아니라 **위계 극단화**(weight 400↔800, 크기 점프, tracking, tabular-nums)로 푼다.
- 알파 합성이 필요하면 리터럴 금지: `color-mix(in oklch, var(--color-*) N%, transparent)` 패턴만.

## 1. PROJECT.md `## 디자인 방향` 해석

| 항목 | 해석 |
|---|---|
| 방향 키워드 | 구체 명사만 유효. "모던함/세련됨/미니멀"이 적혀 있으면 사용자에게 구체화 1회 요청 |
| 레퍼런스 × 차용점 | 짝이 없는 레퍼런스("토스처럼")는 차용점을 1회 되묻기 |
| 시그니처 요소 | 주제의 사물 세계에서 왔는지 검사. 어느 브리핑에나 붙는 장식(blob 등)이면 기각 후 재제안 |
| 프로젝트 전용 금지 | §2 공통 목록에 추가 |
| 다이얼 | 비어 있으면 변주 7 / 모션 4 / 밀도 4 |

섹션 자체가 없으면: §2 기본 방어만 적용하고, 완료 보고에
"PROJECT.md에 `## 디자인 방향`을 채우면 결과가 크게 달라집니다" 한 줄 안내.

## 2. 공통 Anti-Slop 금지 목록 (하드 룰)

### 레이아웃
- ❌ 가운데 정렬 히어로(중앙 헤드라인+서브카피+버튼 2개) — 변주 5 이상에서 금지
- ❌ 균일 3열 카드 그리드 → 2열 지그재그·비대칭(2fr/1fr)·가로 스크롤
- ❌ 빈 셀 있는 벤토 그리드 (span 조합 수학적으로 맞물릴 것)
- ❌ 헤드라인 4줄 이상 줄바꿈 (컨테이너 넓히고 크기 줄여 2~3줄)
- ❌ 모든 섹션 같은 좌/우 교대 반복

### 색
- ❌ 보라/네온 그라데이션, 그라데이션 blob ("AI 퍼플" 일체)
- ❌ 순수 검정 #000 (neutral-950 계열)
- ❌ 액센트 2색 이상 / 채도 80% 초과
- ❌ 대형 헤드라인 그라데이션 텍스트
- ❌ 토큰 밖 모든 색값 (훅이 차단 — hex/rgb/hsl/oklch 리터럴)

### 타이포
- ❌ 크기로만 위계 (weight·색·tracking 조합 없이)
- ❌ 본문 65ch 초과 행길이
- ❌ 의미 없는 번호("01/02/03")·메타 라벨("SECTION 01") — 실제 순서가 정보일 때만

### 콘텐츠·카피
- ❌ 이모지 아이콘 (일관 stroke의 SVG로)
- ❌ "홍길동/John Doe" 더미명, "99.9%/10,000+" 깔끔한 가짜 숫자 (유기적 수치: 47.2%, 1,284)
- ❌ AI 카피 클리셰: "Seamless", "혁신적인", "스마트한", "~의 모든 것"

### 질감·모션
- ❌ 기본 linear/ease-in-out (custom cubic-bezier — 예: 0.16,1,0.3,1)
- ❌ 진한 무채색 drop-shadow → 배경 hue로 틴트된 부드러운 그림자
- ❌ 산만한 다발 애니메이션 → 오케스트레이션된 1개 모먼트
- ❌ prefers-reduced-motion 무시 / transform·opacity 외 속성 애니메이션
- ✅ (의무) 화면당 **관습을 깨는 결정 1개** + 한 줄 근거를 완료 보고에 포함

### AI 기본 룩 3종 경계
브리핑이 명시 안 했는데 다음으로 수렴하면 기각·재설계:
① 크림 배경+고대비 세리프+테라코타 ② 거의-검정+형광 단일 액센트 ③ 헤어라인+radius 0 신문 브루탈리즘

## 3. 시리즈 일관성 규칙 (v0.8 — 데모 2종 제작에서 도출)

한 화면이 아니라 **화면 시리즈**를 만들 때만 보이는 규칙:

1. **시그니처는 재발명하지 않고 번역한다.** 같은 사물 언어의 다른 활용.
   예: 보딩패스의 ICN→KIX 구간 → 예약 화면의 체크인→체크아웃 구간. 절취선은 "확정 직전" 의미로 재등장.
2. **같은 의미에 액센트색 중복 금지.** 화면당 액센트의 일은 하나.
   예: 날짜 선택 상태가 코랄이면, 시간 칩 선택 상태는 neutral-900 — 둘 다 코랄이면 강조 위계 붕괴.
3. **사진 처리 통일.** 필터(예: grayscale 0.35 + contrast 1.1)는 방향의 일부 — 화면마다 다르면 다른 앱처럼 보인다.
4. **수치 표기 통일.** tabular-nums, 날짜 포맷(6.18), 통화 포맷을 시리즈 전체에서 고정.

## 4. 템플릿 답안 기각 절차 (의무)

코드 작성 전, 이 화면 유형의 **통계적 평균 답안을 한 줄로 명시하고 기각**한다.
- 여행 메인: "인사말+검색바+카테고리 아이콘 행+인기 카드 그리드" → 기각
- 예약: "이미지 캐러셀+별점 행+편의시설 아이콘 그리드+하단 고정 가격바" → 기각
기각문이 없으면 설계를 시작하지 않는다. (평균을 인식해야 평균 밖으로 나간다)

## 5. 완료 게이트 (ds-screen-designer가 따름)

- [ ] 템플릿 답안 기각문 1줄
- [ ] 시그니처 번역 여부 (시리즈 2번째 화면부터)
- [ ] 색 페어 전수 check.mjs PASS (`--fg/--bg` 직접 검증 또는 토큰 파일 검사)
- [ ] 토큰 규율: :root/토큰 파일 밖 raw 색값 0 (훅+eval T2가 이중 차단)
- [ ] 스토리 생성 → `pnpm test:a11y` 검사망 편입
- [ ] 관습 파괴 1개 + 근거 한 줄
- [ ] prefers-reduced-motion 분기 존재
